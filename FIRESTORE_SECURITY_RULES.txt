FIRESTORE SECURITY RULES
========================

Copy and paste these rules into your Firebase Console:
https://console.firebase.google.com/project/gulistan-store/firestore/rules

---

rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /expenses/{document} {
      allow read, write: if true;
    }
    match /investments/{document} {
      allow read, write: if true;
    }
    match /suppliers/{document} {
      allow read, write: if true;
    }
    match /invoices/{document} {
      allow read, write: if true;
    }
    match /exchange/{document} {
      allow read, write: if true;
    }
    match /cash/{document} {
      allow read, write: if true;
    }
    match /customers/{document} {
      allow read, write: if true;
    }
    match /payments/{document} {
      allow read, write: if true;
    }
    match /outstanding/{document} {
      allow read, write: if true;
    }
    match /outstanding_payments/{document} {
      allow read, write: if true;
    }
    match /customer_payments/{document} {
      allow read, write: if true;
    }
  }
}

---

NOTES:
======
1. outstanding_payments - Main collection for tracking payments against outstanding items
2. customer_payments - Alternative name candidate for Firebase cloud storage

These rules allow full read/write access (allow read, write: if true;)
For production, consider restricting to authenticated users:

  allow read, write: if request.auth != null;

Or restrict by specific claims/roles for better security.
